Methodology

4 tasks × 3 runs × 5 environments = 60 sessions. Each task was run 3 times per environment to measure variance. We report the minimum, average, and maximum token counts. Total tokens = session tokens + subagent tokens, representing the true consumed cost per run. All sessions ran on Claude Opus 4.6 (1M context) via Claude Code v2.1.76–v2.1.77.

Analysis
Key Findings

Patterns that emerged from the 60-session benchmark.

▼
CodeGraph eliminates subagent overhead on simple tasks
On Task 1, CodeGraph used 0 subagents and ~29K tokens vs. Baseline's 1 subagent + 67K subagent tokens. By providing symbol-level navigation, CodeGraph made exploration subagents unnecessary — the agent found what it needed directly.
⚠
Context Mode inflates session tokens significantly
Context Mode (E4) had the highest average session tokens across all tasks. The MCP tools (ctx_batch_execute, ctx_search, ctx_fetch_and_index) inject large context payloads into the session, causing 40–110% more tokens than Baseline on Tasks 1–2. The tradeoff is zero subagent usage.
⇵
Deterministic Commands add consistency, not savings
E3 showed nearly identical token counts to Baseline but guaranteed lint and type-check runs. Every session had at least 1 tsc + 1 eslint run. The value is reliability (catching errors before delivery), not token reduction.
▲
Complex tasks narrow the gap between strategies
On Task 3 (22-file migration) and Task 4 (greenfield feature), all environments converge toward 55K–75K session tokens. The bulk of the work — reading files, writing code, editing — dominates regardless of exploration strategy. Optimization strategies have diminishing returns on large tasks.

Recommendation

CodeGraph + Deterministic Commands (E5) offers the best overall value proposition. It combines CodeGraph's subagent-elimination benefit with guaranteed verification steps. Across all tasks, E5 averaged the lowest or second-lowest total effective tokens while maintaining full lint/type-check coverage on every run.

Context Mode is valuable when external documentation lookup is required (e.g., migration guides), but its higher session token overhead makes it less suitable as a default. Consider enabling it selectively for tasks that involve third-party API/library changes.
